### Installed Components

- **Sysmon** – Monitors process creation, network connections, and other system events.
- **Configuration File** – Custom XML config applied to filter and log relevant events.
- **Windows Event Log** – Sysmon logs are written to `Microsoft-Windows-Sysmon/Operational`.

---

### Key Sysmon Event IDs

| Event ID | Description |
|----------|-------------|
| **1**    | Process creation – Captures process name, command-line arguments, and parent process |
| **3**    | Network connection – Records source/destination IPs, ports, and protocols |
| **10**   | Process access – Logged when one process accesses another (useful for post-login activity analysis) |

---

### Configuration Highlights

- Only processes relevant to credential attacks are monitored (RDP, SMB, Powershell, CMD, etc.)
- Network connections are logged for all inbound/outbound activity.
- Sysmon events are filtered to reduce noise while maintaining attack visibility.
- Logs are forwarded to the SIEM using **Splunk Universal Forwarder** (covered in 03.03).
