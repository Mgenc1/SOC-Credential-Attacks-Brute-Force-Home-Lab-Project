
## 1. Architecture

The log forwarding process follows this flow:

Kali Linux (Attacker) → Windows 11 (Victim) → Splunk SIEM (Ubuntu Server)

Attack traffic generated from the attacker machine is logged on the Windows victim system.  
These logs are then forwarded to the Splunk server for indexing and analysis.

| Machine        | Role           | IP Address      |
|---------------|----------------|-----------------|
| Kali Linux    | Attacker       | 192.168.1.106   |
| Windows 11    | Victim         | 192.168.1.107   |
| Ubuntu Server | Splunk SIEM    | 192.168.1.105   |

---

## 2. Installing Splunk Universal Forwarder

1. Download **Splunk Universal Forwarder for Windows** from the official Splunk website.
2. Install the forwarder on the Windows victim machine.
3. During installation, configure the **Splunk server IP**: `192.168.1.105`

This allows the forwarder to send logs directly to the SIEM server.

---

## 3. Configuring Log Inputs

The **inputs.conf** configuration file defines which logs are forwarded to Splunk.

### Example Configuration

```ini
[WinEventLog://Security]
index = windows
disabled = 0

[WinEventLog://Microsoft-Windows-Sysmon/Operational]
index = sysmon
disabled = 0
````

### Logs Collected

This configuration ensures that:

* **Windows Security Logs** (authentication, privilege events)
* **Sysmon Logs** (process, network, and memory activity)

are continuously forwarded to Splunk.

---

## 4. Log Transmission (Real Evidence)

Once configured, the Universal Forwarder continuously sends logs from the Windows machine to Splunk.

### Example Forwarded Security Log

```text
EventCode=4625
Account Name=testuser
Source Network Address=192.168.1.106
Logon Type=3
Authentication Package=NTLM
```

### Example Forwarded Sysmon Log

```text
EventCode=3
SourceIp=192.168.1.106
DestinationIp=192.168.1.107
DestinationPort=3389
Image=C:\Windows\System32\svchost.exe
```

### Example Process Creation (Forwarder Activity)

```text
EventCode=1
Image=C:\Program Files\SplunkUniversalForwarder\bin\splunk-powershell.exe
ParentImage=C:\Program Files\SplunkUniversalForwarder\bin\splunkd.exe
User=NT AUTHORITY\SYSTEM
```

These logs confirm that:

* Attack traffic is successfully captured on the Windows host
* Sysmon provides network-level visibility (RDP / SMB activity)
* Splunk Forwarder is actively collecting and transmitting logs

---

## 5. Verification in Splunk

To confirm that logs are successfully received:

```spl
index=windows OR index=sysmon
| stats count by sourcetype
```

### Expected Results

* `WinEventLog:Security`
* `WinEventLog:Microsoft-Windows-Sysmon/Operational`

---

## 6. Advanced Validation (Correlation Check)

To verify attack visibility:

```spl
index=windows EventCode=4625 OR EventCode=4624
| stats count by EventCode, Source_Network_Address, Account_Name
```

### ✔ Expected Finding

* Multiple **4625 (failed logons)** from `192.168.1.106`
* Followed by a **4624 (successful logon)** from the same IP

---

## 7. Security Insight

Based on forwarded logs:

* Repeated failed logons (**4625**) from attacker machine
* Successful authentication (**4624**) using same credentials
* Sysmon logs confirm:

  * RDP traffic → **Port 3389**
  * SMB traffic → **Port 445**
* Process access to **lsass.exe** detected (**Sysmon Event ID 10**)

The logging pipeline is successfully configured and validated:
