## 1. Step 1 – Verify Data Ingestion

The first step is to confirm that logs are arriving in Splunk.

Run the following search query in Splunk:

```spl
index=windows OR index=sysmon
| stats count by sourcetype
```

If the configuration is correct, Splunk will display event counts for the following log sources:

Windows Security Logs

Sysmon Operational Logs

This confirms that the Universal Forwarder is successfully sending data to Splunk.

---

## 2. Step 2 – Validate Authentication Events

Next, verify that Windows authentication logs are being collected.

```spl
index=windows EventCode=4625
| table _time ComputerName Account_Name Source_Network_Address
| sort - _time
```

This search displays failed login attempts, which are commonly generated during brute force or password spraying attacks.

Expected fields include:

Timestamp

Computer name

Account name

Source IP address

Example attack source: Source_Network_Address: 192.168.1.106

---

## 3. Step 3 – Validate Sysmon Logs

To confirm that Sysmon logs are also arriving in Splunk, run:

```spl
index=sysmon
| stats count by EventCode
```

This query verifies the presence of Sysmon events such as:

| Event ID | Description        |
| -------- | ------------------ |
| 1        | Process Creation   |
| 3        | Network Connection |
| 10       | Process Access     |

These events provide additional visibility into system activity during attack simulations.

---

## 4. Step 4 – Confirm Log Correlation

Finally, confirm that attack activity generates correlated logs.

Example correlation scenario:

Kali attacker performs an RDP brute force attack.

Windows logs Event ID 4625 (Failed Login).

Sysmon logs Event ID 3 (Network Connection).

Both events appear in Splunk.

This confirms that the entire attack → logging → SIEM pipeline is functioning correctly.

---

## 5. Validation Result

The log validation process confirms that:

Windows Security logs are successfully forwarded to Splunk

Sysmon operational logs are successfully forwarded

Authentication events generated during attack simulations are visible

The SIEM platform is ready for SOC monitoring and attack detection
