
### Key Authentication Event IDs

| Event ID | Description | SOC Relevance |
|----------|-------------|---------------|
| **4625** | Failed logon attempt | Indicates possible brute force or password spraying activity |
| **4624** | Successful logon | Shows successful authentication after multiple failures |
| **4740** | Account lockout | Triggered when login attempts exceed lockout policy |
| **4672** | Privileged logon | Indicates login with administrative privileges |
| **4720** | New user account created | May indicate persistence or unauthorized account creation |
| **4728** | User added to a security-enabled group | Possible privilege escalation |

---

### Example Security Event

Example **Event ID 4625 – Failed Logon Attempt**

```text
Event ID: 4625
Logon Type: 3
Account Name: victimuser
Source Network Address: 192.168.1.106
Failure Reason: Unknown user name or bad password
```

Example Event ID 4624 – Successful Logon

```text
Event ID: 4624
Logon Type: 10
Account Name: victimuser
Source Network Address: 192.168.56.106
Authentication Package: Negotiate
```
---
### Detection Indicators

SOC analysts monitor these logs for patterns such as:

Multiple Event ID 4625 entries from the same IP address

A 4624 successful login occurring after multiple failures

Sudden account lockouts (4740)

Logins with administrative privileges (4672)

These indicators help identify RDP brute force attacks, SMB authentication attempts, and password spraying campaigns.
