

## 1. Key Authentication Event IDs

````

| Event ID | Description | SOC Relevance |
|----------|-------------|---------------|
| **4625** | Failed logon attempt | Indicates possible brute force or password spraying activity |
| **4624** | Successful logon | Shows successful authentication, especially after multiple failures |
| **4740** | Account lockout | Triggered when login attempts exceed lockout policy |
| **4672** | Privileged logon | Indicates login with administrative privileges |
| **4720** | New user account created | May indicate persistence or unauthorized account creation |
| **4728** | User added to a security-enabled group | Possible privilege escalation |

````

## 3. Detection Indicators

SOC analysts monitor these logs for the following attack patterns:

* Multiple **Event ID 4625** entries from the same source IP (e.g., 192.168.1.106)
* A **successful login (4624)** occurring shortly after repeated failures
* Same **source IP + same user correlation** (strong compromise indicator)
* Logon Type differences:

  * **Type 10 → RDP (RemoteInteractive)**
  * **Type 3 → SMB / Network logon**
* Presence of **NTLM authentication** during attack attempts
* Privileged logon events (**4672**) following authentication
* Sysmon Event ID 3 → Network connections to:

  * **3389 (RDP)**
  * **445 (SMB)**
* Sysmon Event ID 10 → Access to **lsass.exe** (possible credential dumping attempt)

## 4. Attack Pattern Observed (From Logs)

The following sequence was identified:

1. Multiple failed logon attempts (**4625**) from:

   * Source IP: **192.168.1.106**
   * Target user: **testuser**

2. A successful logon (**4624**) shortly after:

   * Same IP: **192.168.1.106**
   * Same user: **testuser**

3. Sysmon logs confirm:

   * Network connections to **RDP (3389)** and **SMB (445)**
   * Process interaction with **lsass.exe**

---

## 5. Conclusion

This behavior is consistent with:

* **Brute Force Attack**
* **Credential Compromise**
* Potential **Lateral Movement**

The correlation between failed and successful logons, combined with network and process activity, confirms malicious authentication attempts originating from **192.168.1.106**.
