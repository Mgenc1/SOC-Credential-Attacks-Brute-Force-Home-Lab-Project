## 1. Failed Login Thresholds

**Logic:** Trigger an alert when failed logins exceed a configurable threshold within a set time window.

```text
Rule: Failed login attempts > 3 within short time
Trigger: Alert in Splunk
Action: Correlate IP & user; escalate if threshold is met
```

**Explanation:**

* Detects **brute-force attacks** early
* Avoids false positives from occasional mistyped passwords
* Example: 192.168.1.106 attempts login 7 times in 5 minutes → flagged

---

## 2. Successful Login After Failures

**Logic:** Correlate successful logins (Event ID 4624) that follow multiple failures (Event ID 4625).

```text
Condition: Failed login count ≥ 3 followed by a success within short time
Action: Flag as potential compromise
```

**Explanation:**

* Highlights **accounts that may have been compromised**

---

## 3. Account Lockout & Repeated Attempts

**Logic:** Monitor account lockouts (Event ID 4740) and repeated login failures across multiple accounts.

```text
Condition: Multiple accounts fail from same source IP
Action: Escalate alert, consider blocking IP
```

**Explanation:**

* Detects **password spraying attacks**
* Helps prevent **lateral movement attempts**
* Integrates with **Tier1 triage process** for immediate response

---

## 4. Post-login Behavior Correlation

**Logic:** Monitor suspicious process executions after successful logins.

```text
Condition: High-risk processes created after successful login
Examples: lsass.exe access, powershell scripts, credential dumping tools
Action: Trigger IOC alert & add to MITRE mapping
```

**Explanation:**

* Detects **post-compromise activity** (T1059, T1003)
* Correlates with **user & source IP**
* Critical for **early containment and investigation**

---

## 5. Anomaly Detection Enhancements

* Track **login times** vs normal user behavior
* Identify **unusual source IPs or geolocations**
* Monitor **privilege escalation events** after login
* Combine **Sysmon & Windows logs** for comprehensive correlation
