## 1. RDP Brute Force Attack Execution

In this scenario, the attacker attempts to gain remote access to the Windows victim machine through the Remote Desktop Protocol (RDP) service.

The attack originates from:

- **Source IP:** 192.168.1.106  
- **Target System:** 192.168.1.107  
- **Target Account:** testuser  

The attacker uses automated tools to perform repeated authentication attempts.

### 1.1. Execution Process

1. The attacker identifies that the RDP service (Port 3389) is enabled.
2. A password list is prepared.
3. Automated login attempts are sent to the RDP service.
4. Failed attempts generate **Windows Event ID 4625**.
5. A successful attempt generates **Event ID 4624**.

### 1.2. Log Evidence

- **4625 (Failed Logon)**
  - Source IP: 192.168.1.106  
- **Sysmon Event ID 3**
  - Destination Port: **3389 (RDP)**  

 Confirms brute force activity over RDP.

## 2. SMB Brute Force Attack Execution

In this scenario, the attacker targets the SMB service on the Windows victim machine.

SMB operates on **Port 445** and is commonly abused for credential attacks.

### 2.1. Execution Process

1. The attacker identifies SMB service exposure.
2. Username and password lists are prepared.
3. Automated authentication attempts are sent to SMB.
4. Failed attempts generate **Event ID 4625**.
5. A successful login generates **Event ID 4624**.

### 2.2. Log Evidence

- **4625 (Failed Logon)**
  - Logon Type: 3 (Network)
  - Source IP: 192.168.1.106  
- **4624 (Successful Logon)**
  - Account: testuser  
  - Source IP: 192.168.1.106  
- **Sysmon Event ID 3**
  - Destination Port: **445 (SMB)**  

 Confirms successful brute force compromise over SMB.

## 3. Log Generation During Attack Simulation

During the brute force attack simulations, the Windows victim machine generates multiple security events.  
These logs are forwarded to Splunk for centralized analysis.

---

## 4. Windows Security Event Logs (Observed)

Based on the lab results, the following events were **actually observed**:

- **Event ID 4625** – Failed logon attempts  
- **Event ID 4624** – Successful logon  
- **Event ID 4672** – Special privileges assigned (observed but not directly attack-related)  
- **Event ID 4648** – Logon using explicit credentials (background/system activity)  
- **Event ID 4634** – Logoff event  

### 4.1. Important Note

Events such as below were **NOT observed in this lab**, but may appear in real environments:

- Event ID 4740 – Account lockout  
- Event ID 4720 / 4728 – User creation / privilege changes  

These depend on domain configuration and attack behavior.

---

## 5. Sysmon Event Logs (Observed)

Sysmon provides deeper visibility into system and network activity:

- **Event ID 3** – Network connection  
  - Shows attacker IP connecting to:
    - Port 3389 (RDP)
    - Port 445 (SMB)

- **Event ID 1** – Process creation  
  - Observed from Splunk Forwarder (normal activity)

- **Event ID 10** – Process access  
  - Access to **LSASS** observed (requires context-based investigation)

---

## 6. Key Observation

The attack is clearly visible through **log correlation**:

1. Multiple **4625 events** (failed logins)  
2. A **4624 event** (successful login)  
3. Sysmon **Event ID 3** (network connection to target service)  
4. Same source IP: **192.168.1.106**

---

## 7. Validation Result

- Brute force attempts successfully generate authentication logs  
- Successful compromise is recorded in Event ID 4624  
- Sysmon confirms network-level attack activity  
- Logs are correctly forwarded to Splunk for analysis  

---

## 8. Key Takeaway

This simulation demonstrates that effective detection requires:

- Authentication visibility (Windows logs)
- Network visibility (Sysmon)
- Correlation across multiple data sources

Single log sources are insufficient — **correlation is critical in SOC investigations**.
