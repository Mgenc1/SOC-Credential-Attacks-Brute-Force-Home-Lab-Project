
## Tools Used

The following offensive security tools were used during the attack simulations:

- **Hydra** – Used for automated brute force attacks against RDP and SMB services
- **CrackMapExec** – Used for SMB authentication testing and password spraying simulations

These tools generate multiple authentication attempts that trigger security events on the Windows victim machine.

---

## RDP Brute Force Attack

Hydra was used to perform a brute force attack against the Remote Desktop Protocol (RDP) service running on the Windows victim machine.

Example command:

```bash
hydra -l victimuser -P passwords.txt rdp://192.168.56.110
```
Explanation:

-l specifies the target username

-P specifies the password wordlist

rdp:// indicates the RDP protocol target

192.168.56.110 is the IP address of the Windows victim machine

This command attempts multiple password combinations against the specified user account.

Each failed attempt generates Windows Event ID 4625, while a successful login generates Event ID 4624.

---
## SMB Brute Force Attack

Hydra can also be used to perform brute force authentication attempts against the SMB service on the Windows machine.

Example command:
```bash
hydra -l victimuser -P passwords.txt smb://192.168.56.110
```
This attack targets the SMB authentication service used for Windows file sharing.

Multiple failed login attempts are generated during the process, which are recorded in Windows Security logs.
---
## Password Spraying Attack

Password spraying attacks were simulated using CrackMapExec.
This technique attempts a single password against multiple user accounts to avoid triggering account lockout policies.

Example command:
```bash
crackmapexec smb 192.168.56.110 -u users.txt -p Winter2024!
```
Explanation:

smb specifies the SMB protocol

-u users.txt contains a list of target usernames

-p specifies the password being tested

192.168.56.110 is the victim machine IP address

The tool attempts authentication across multiple accounts using the same password.
