## 1. Attacker Machine – Kali Linux

The attacker machine represents an external threat actor attempting to gain unauthorized access to the target system. This virtual machine runs **Kali Linux**, a penetration testing distribution commonly used for offensive security operations.

In this lab, Kali Linux is used to perform several credential-based attacks against the victim machine.

### 1.1. Responsibilities

- Launch **RDP brute force attacks** against the Windows host
- Perform **SMB authentication brute force attempts**
- Generate authentication traffic that will later be analyzed in the SIEM

### 1.2. Tools Used

- Hydra
- CrackMapExec

The attacker machine communicates directly with the victim system inside the isolated lab network.

---

## 2. Victim Machine – Windows 11

The victim machine represents a typical enterprise workstation targeted by attackers. This system runs **Windows 11** and hosts the services that will be attacked during the simulation.

To enable security monitoring and forensic analysis, the victim machine is configured with multiple logging mechanisms.

### 2.1. Security Monitoring Components

- **Windows Security Event Logs** – capture authentication events such as login attempts
- **Sysmon** – records detailed system activity such as process creation and network connections
- **Splunk Universal Forwarder** – forwards logs to the SIEM server for centralized monitoring

### 2.2. Events Generated During Attacks

During the brute force simulations, the system generates several important security events, including:

- Failed login attempts
- Successful authentications
- Account lockouts
- Privileged logins
- Post-login process activity

These logs serve as the primary evidence used for detection and investigation.

---

## 3. SIEM Machine – Ubuntu Server (Splunk)

The SIEM machine acts as the **central monitoring and analysis platform** for the lab environment. This system runs **Splunk Enterprise** on an Ubuntu Server.

Its main role is to collect, index, and analyze logs generated by the victim machine.

### 3.1. Capabilities

The SIEM platform allows analysts to:

- Monitor authentication activity in real time
- Detect brute force attack patterns
- Correlate security events across multiple log sources
- Identify Indicators of Compromise (IOCs)
- Build dashboards and alerts for suspicious behavior

Logs from the Windows machine are forwarded to the SIEM using the **Splunk Universal Forwarder**, allowing analysts to investigate attack activity through **Splunk search queries and dashboards**.

---

## 4. Lab Communication Flow

The interaction between the machines follows a simple structure:

1. The **Kali attacker machine** initiates credential-based attacks.
2. The **Windows victim machine** processes authentication attempts and records security events.
3. These logs are forwarded to the **Splunk SIEM server** for analysis.

This architecture allows the lab to simulate a complete **attack–detection workflow**, similar to the processes used in real **Security Operations Centers (SOCs)**.
