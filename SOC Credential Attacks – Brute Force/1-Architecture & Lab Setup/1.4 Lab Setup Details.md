
## 1. Virtual Machine Setup

### 1.1. Attacker Machine – Kali Linux
- **OS:** Kali Linux (latest stable version)
- **RAM:** 2 GB
- **CPU:** 2 cores
- **Disk:** 20 GB
- **Network Adapter:** Host-only or Internal Network
- **Purpose:** Execute Brute Force attacks (RDP, SMB)
- **Installed Tools:**
  - Hydra
  - CrackMapExec
  - Nmap
- **Additional Configuration:**
  - Enable SSH for remote management (optional)
  - Set static IP: `192.168.1.106`
  - Ensure time synchronization with SIEM for accurate log correlation

---

### 1.2. Victim Machine – Windows 11
- **OS:** Windows 11 Pro
- **RAM:** 2 GB
- **CPU:** 2 cores
- **Disk:** 40 GB
- **Network Adapter:** Host-only or Internal Network (same as Kali)
- **Purpose:** Represent target system; log all authentication and system activity
- **Installed Tools:**
  - Sysmon (Event ID 1, 3, 10)
  - Windows Security Auditing
  - Splunk Universal Forwarder
- **Additional Configuration:**
  - Enable RDP and SMB services for attacks
  - Set static IP: `192.168.1.107`
  - Configure Windows Firewall to allow RDP/SMB within lab network
  - Time synchronization with SIEM

---

### 1.3. SIEM Machine – Ubuntu Server (Splunk)
- **OS:** Ubuntu Server 22.04 LTS
- **RAM:** 4 GB
- **CPU:** 2 cores
- **Disk:** 40 GB+
- **Network Adapter:** Host-only or Internal Network
- **Purpose:** Collect, index, and analyze logs from Victim machine
- **Installed Tools:**
  - Splunk Enterprise
- **Additional Configuration:**
  - Set static IP: `192.168.1.105`
  - Ensure Splunk is reachable from Victim and Attacker machines
  - Configure indexes and dashboards for authentication activity

---

## 2. Network Setup

- **Network Type:** Host-only / Internal Network (isolated lab)
- **Subnet:** `192.168.1.0/24`
- **IP Assignments:**
  - Kali (Attacker): `192.168.1.106`
  - Windows 11 (Victim): `192.168.1.107`
  - Ubuntu Server (SIEM): `192.168.1.105`
- **Traffic Flow:**
  1. Attacker sends RDP/SMB authentication attempts to Victim
  2. Victim logs activity via Sysmon and Windows Security
  3. Logs forwarded to SIEM for collection, analysis
- **NAT/Firewall:**
  - No external internet connection required
  - Internal firewall rules may allow only lab traffic
  - SIEM firewall allows incoming traffic from Victim on Splunk port (default: 9997)

---

## 3. Additional Notes

- Synchronize clocks on all VMs for accurate log correlation
- Take snapshots before attack simulations to revert if needed
- Lab must remain fully isolated to prevent accidental exposure
