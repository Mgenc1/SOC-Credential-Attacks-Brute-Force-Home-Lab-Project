This section provides detailed instructions on how to configure each virtual machine, the network environment, and firewall/NAT settings for the SOC Brute Force Credential Attack Lab. Proper setup ensures realistic simulations and reliable log collection.

Virtual Machine Setup
1. Attacker Machine – Kali Linux

OS: Kali Linux (latest stable version)

RAM: 4 GB

CPU: 2 cores

Disk: 20 GB

Network Adapter: Host-only or Internal Network

Purpose: Execute Brute Force attacks (RDP, SMB) and Password Spraying.

Installed Tools:

Hydra

CrackMapExec

Nmap (for network reconnaissance)

Additional Configuration:

Enable SSH for remote management if needed.

Set static IP: 192.168.1.106

Ensure time synchronization with SIEM for accurate logs.

2. Victim Machine – Windows 11

OS: Windows 11 Pro

RAM: 4 GB

CPU: 2 cores

Disk: 40 GB

Network Adapter: Host-only or Internal Network (same as Kali)

Purpose: Represent target system; log all authentication and system activity.

Installed Tools:

Sysmon (Event ID 1, 3, 10)

Windows Security Auditing

Splunk Universal Forwarder

Additional Configuration:

Enable RDP and SMB services for attacks.

Set static IP: 192.168.1.107

Disable Windows Firewall or configure to allow RDP/SMB traffic within lab.

Configure time synchronization with SIEM.

3. SIEM Machine – Ubuntu Server (Splunk)

OS: Ubuntu Server 22.04 LTS

RAM: 4–8 GB

CPU: 2 cores

Disk: 40 GB+

Network Adapter: Host-only or Internal Network

Purpose: Collect, index, and analyze logs from Victim machine.

Installed Tools:

Splunk Enterprise

Splunk Universal Forwarder (if forwarding other logs)

Additional Configuration:

Set static IP: 192.168.1.105

Ensure Splunk is reachable from Victim and Attacker machines.

Configure indexes and dashboards for authentication activity.

Network Setup

Network Type: Host-only / Internal Network (isolated lab)

Subnet: 192.168.1.0/24

IP Assignments:

Kali (Attacker): 192.168.1.106

Windows 11 (Victim): 192.168.1.107

Ubuntu Server (SIEM): 192.168.1.105

Traffic Flow:

Attacker sends RDP/SMB authentication attempts to Victim.

Victim logs all activity via Sysmon and Windows Security.

Logs are forwarded to SIEM for collection, visualization, and analysis.

NAT/Firewall:

No external internet required.

Internal firewall rules may be configured on Windows to allow only lab traffic.

SIEM firewall must allow incoming traffic from Victim on Splunk ports (default 9997 or custom).

Additional Notes

Ensure all VMs have synchronized clocks for log correlation.

Take snapshots of VMs before starting attack simulations to revert if needed.

Lab should remain fully isolated to prevent accidental exposure of attacks to production networks.
