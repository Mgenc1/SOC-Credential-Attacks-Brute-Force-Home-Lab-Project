## 1. Incident Response Actions

| Phase | Action | Description |
|------|------|-------------|
| Detection | Alert Validation | Confirm the brute-force activity by correlating multiple failed login events (4625) followed by a successful authentication (4624). |
| Identification | Source Verification | Identify the source host (`192.168.1.106`) responsible for the authentication attempts and verify whether the activity is authorized or malicious. |
| Containment | Block Attacker IP | Block the source IP address at the firewall or endpoint protection system to prevent further authentication attempts. |
| Containment | Account Protection | Temporarily disable or reset the compromised account (`testuser`) to prevent unauthorized access. |
| Investigation | Log Review | Review Windows Security and Sysmon logs to identify additional suspicious activities such as lateral movement or abnormal SMB usage. |
| Investigation | Host Analysis | Examine the affected host (`192.168.1.107`) for unusual processes, unauthorized file access, or persistence mechanisms. |
| Eradication | Credential Reset | Enforce password resets for the compromised account and any related accounts. |
| Recovery | System Monitoring | Continue monitoring authentication logs and network activity to ensure the attack has been fully mitigated. |

---

## 2. Preventive Security Measures

| Control | Description |
|------|-------------|
| Account Lockout Policy | Configure account lockout thresholds to automatically block repeated failed login attempts. |
| Multi-Factor Authentication (MFA) | Require MFA for remote authentication to reduce the risk of credential-based attacks. |
| Network Segmentation | Restrict SMB access between systems where possible to limit lateral movement. |
| Log Monitoring | Implement SIEM detection rules to identify brute-force patterns in authentication logs. |

> **Operational Note:** Implementing account lockout policies, enforcing MFA, and monitoring authentication anomalies can significantly reduce the effectiveness of brute-force attacks targeting SMB services.
