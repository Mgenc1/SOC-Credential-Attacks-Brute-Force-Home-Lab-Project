### Scenario Overview
This case simulates an SMB brute force attack where an external host attempts to gain unauthorized access to the target machine by trying multiple username and password combinations over the SMB protocol. The attack leverages network logon requests (Logon Type 3) to test credentials on the remote system.

**Case:** SMB Brute Force  
**Date:** 13 March 2026  
**Host:** `umut` (Windows 11)  
**Attacker IP:** 192.168.1.106  
**Target Service:** SMB (TCP 445) 

---

### Objective
- Detect and correlate failed and successful logon attempts.  
- Identify attacker's source IP and pattern of connections.  
- Verify the creation of network connections and process executions associated with the attack.  
- Gather evidence for incident reporting and mitigation actions.

---

### Attack Steps
1. Attacker scans the target network for SMB service on TCP port 445.  
2. Multiple login attempts are made using brute force tools (`hydra`, `crackmapexec`, etc.) or manual credential testing.  
3. Successful logon occurs once correct credentials are found.  
4. All activity is captured via:
   - **Windows Security Logs:** 4625 (failed logon), 4624 (successful logon)  
   - **Sysmon Logs:** Event 3 (network connection), Event 1 (process creation)  

---

### Notes
- Logon Type 3 indicates network logon attempts, which are typical for SMB brute force attacks.  
- Event 3 in Sysmon confirms that TCP 445 connections were initiated from the attacker host.  
- Event 1 (Process Creation) can help identify local processes that executed the attack tools, though in this simulation `splunk-powershell.exe` is shown as a placeholder.
