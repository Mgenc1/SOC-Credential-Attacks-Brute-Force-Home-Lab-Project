### 1. Scenario Overview

An attacker attempts to gain unauthorized access by **systematically guessing passwords** for a target user account through the RDP service. This attack mimics a **realistic brute force attempt** that SOC analysts could encounter in production.

- **Attacker:** Kali Linux machine (`192.168.1.106`)  
- **Target:** Windows 11 victim machine (`192.168.1.107`)  
- **Service Targeted:** RDP (TCP 3389)  
- **Detection Tool:** Splunk SIEM (`192.168.1.105`) via Splunk Universal Forwarder  

---

### 2. Attack Flow

1. The attacker performs multiple authentication attempts within a short period.  
2. Each **failed login** triggers a **Windows Security Event ID 4625**.  
3. If correct credentials are guessed, a **successful login event (Event ID 4624)** is generated.  
4. Sysmon logs capture additional post-login activity (Event IDs 1, 3, 10), such as process creation and LSASS access.  
5. All events are forwarded to **Splunk SIEM** for analysis and alerting.

---

### 3. SOC Objectives

The purpose of this scenario is to:

- Simulate a realistic **RDP brute force attack** in a controlled environment  
- Observe **Windows Security and Sysmon events** generated by failed and successful logins  
- Allow SOC analysts to **detect, triage, and investigate** brute force activity  
- Provide logs for **timeline analysis, IOC extraction, and threat scoring**
