This case study simulates a **Remote Desktop Protocol (RDP) brute force attack** against the Windows 11 victim machine in the SOC lab environment.

In this scenario, an attacker attempts to gain unauthorized access by repeatedly trying different password combinations for a target user account through the RDP service.

The attack originates from the **Kali Linux attacker machine (192.168.1.106)** and targets the **Windows 11 victim machine (192.168.1.107)** where the RDP service is enabled.

During the attack, multiple authentication attempts are generated within a short period of time.  
Each failed login attempt triggers a **Windows Security Event ID 4625**, which is recorded in the Windows Security log.

If the attacker eventually guesses the correct credentials, the system will generate a **successful authentication event (Event ID 4624)**.

These events are forwarded to the **Splunk SIEM server (192.168.1.105)** through the Splunk Universal Forwarder, allowing SOC analysts to detect and investigate the brute force activity.

The objective of this case study is to:

- Simulate a realistic RDP brute force attack
- Observe authentication events generated on the Windows system
- Detect attack activity using Splunk SIEM
- Perform SOC Tier 1 alert triage and investigation
