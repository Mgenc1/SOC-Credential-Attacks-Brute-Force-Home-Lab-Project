## 1. Objective

Identify the top source IP addresses generating failed login attempts in the environment.
This helps detect brute-force attacks and suspicious activity.

---

## 2. Data Source

* Windows Security Logs
* Event ID:

  * **4625** → Failed Logon
* Collected via:

  * Splunk Universal Forwarder

---

## 3. Sample Log Evidence

From collected logs:

* Source IP: **192.168.1.106**
* EventCode: **4625**
* Logon Type: **3 (Network)**
* Username Attempted: **testuser**

This indicates a brute-force attempt from attacker machine.

---

## 4. Splunk Query

```spl
index=main EventCode=4625
| stats count by Source_Network_Address
| sort -count
| head 10
```

---

## 5. Visualization

* Chart Type: **Bar Chart**
* X-Axis: `Source_Network_Address`
* Y-Axis: `count`

---

## 6. Analysis

* The IP address **192.168.1.106** generated multiple failed login attempts.

* This behavior is consistent with:

  * SMB brute force
  * RDP brute force

* Logon Type 3 confirms:

  * Remote network-based authentication attempts

---

## 7. Detection Insight

High number of failed logins from a single IP may indicate:

* Brute-force attack
* Credential stuffing
* Unauthorized access attempts

---

## 8. Result

The dashboard successfully identifies:

* Top attacking IPs
* Attack frequency
* Source of brute-force activity
