## 1. Data Source

* Log Source:

  * Windows Security Logs (for correlation)
  * Sysmon Operational Logs
* Collection Method:

  * Splunk Universal Forwarder
* Relevant Event IDs:

  * **4624** – Successful Logon (Security)
  * **Sysmon Event ID 1** – Process Creation
  * **Sysmon Event ID 3** – Network Connections

---

## 2. Analytical Approach

After a successful login (Event ID 4624), attackers typically execute commands or launch tools to maintain access, escalate privileges, or move laterally.

This panel focuses on identifying:

* Processes spawned after login
* Unusual or suspicious binaries
* Parent-child process relationships
* Network connections initiated post-login

---

## 3. Splunk Query (Process Creation Activity)

```spl
index=sysmon EventCode=1
| stats count by Image, User
| sort - count
| head 15
```

---

## 4. Correlation with Successful Logon (Advanced)

```spl
index=main EventCode=4624
| rename _time as login_time, Account_Name as user
| join user [
    search index=sysmon EventCode=1
    | rename _time as process_time, User as user
]
| where process_time >= login_time
| stats count by user, Image
| sort - count
```

---


## 5. Findings

Analysis of Sysmon logs indicates process execution activity following successful authentication.

Observed processes include:

* `services.exe` (system process)
* `svchost.exe` (normal background activity)
* `splunk-powershell.exe` (log forwarding activity)

Additionally, network connections were observed:

* Source IP: **192.168.1.106**
* Destination Port: **3389 (RDP)** and **445 (SMB)**

This confirms post-authentication interaction between attacker and target system.

---

## 6. Detection Insight

Post-login process activity may indicate:

* Command execution by attacker
* Persistence mechanisms
* Lateral movement attempts
* Use of administrative tools

Indicators of concern include:

* Unusual process names
* Command-line execution (PowerShell, cmd.exe)
* Processes spawned by unexpected parent processes

## 8. Advanced Detection

Detect suspicious processes:

```spl
index=sysmon EventCode=1
| search Image="*powershell*" OR Image="*cmd*" OR Image="*psexec*"
| stats count by Image, User
| sort - count
```
