## 1. Incident Context

Based on previous analysis:

- Multiple failed logon attempts (Windows Event ID 4625)
- Followed by a successful logon (Windows Event ID 4624)
- Source IP identified: 192.168.1.106
- Target account: testuser
- Sysmon Event ID 3 confirms the network connection:
  - Process: svchost.exe
  - Destination port: 3389 (RDP)
  - Protocol: TCP

---

## 2. Containment Strategy

### 1. Account Lockout / Disablement

Immediately restrict access to the compromised account.

#### Actions:

* Disable or lock the affected account (**testuser**)
* Force password reset
* Invalidate active sessions

#### Purpose:

* Prevent attacker from maintaining access
* Stop further misuse of compromised credentials

---

### 2. Source IP Blocking

Block the attacker’s IP address at the network level.

#### Actions:

* Block IP: **192.168.1.106**
* Apply rule on:

  * Firewall
  * Endpoint protection system
  * SIEM (for alerting and monitoring)

#### Purpose:

* Prevent further connection attempts
* Stop repeated brute-force activity

---

### 3. Multi-Factor Authentication (MFA) Enforcement

Enable MFA for affected and high-risk accounts.

#### Actions:

* Enforce MFA on:

  * Remote access (RDP, VPN)
  * Privileged accounts
* Apply conditional access policies

#### Purpose:

* Mitigate risk of credential-based attacks
* Add an additional authentication layer

---

## 3. Optional Immediate Actions

* Terminate suspicious sessions
* Review recent login activity for lateral movement
* Isolate affected host if necessary

---

## 4. Supporting Evidence

Containment actions are based on:

* **06.01 Top Attacking IPs** → Identified attacker source
* **06.02 Login Trend** → Detected brute-force pattern
* **06.03 Targeted Users** → Identified compromised account
* **06.05 Post-login Activity** → Observed post-compromise behavior

---

## 5. Risk Consideration

Failure to apply containment may result in:

* Persistent unauthorized access
* Privilege escalation
* Lateral movement within the network

Risk Level: **High**

---

## 6. Outcome

Effective containment ensures:

* Immediate disruption of attacker activity
* Protection of compromised accounts
* Reduction of further attack surface
